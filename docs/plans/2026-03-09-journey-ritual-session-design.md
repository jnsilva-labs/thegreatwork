# Journey Ritual Session Redesign

Date: 2026-03-09
Status: Approved (brainstorming design)
Owner: Awareness Paradox

## 1. Purpose

Redesign `/journey` from a geometry-forward reference page into a guided ritual session that functions as an actual meditative practice surface.

The new page should:
- feel quiet, intentional, and ceremonial
- guide the visitor through one principle at a time
- use sacred geometry as a focal object rather than a dominant overlay
- create an embodied experience instead of another reading surface
- become a distinctive practice destination within the Awareness Paradox ecosystem

## 2. Core Decision (Locked)

`/journey` should primarily be a guided meditation session, not a self-paced contemplative library.

Reason:
- the site already has significant library energy
- the current gap is embodied practice
- a guided ritual session creates more transformation and stronger differentiation

Future secondary layer:
- a contemplative library may be added later below the session or on a separate route such as `/practice` or `/study`

## 3. Diagnosis of the Current Page

The current page is structurally a gallery viewer with principle notes attached.

### Current strengths
- the principle-to-geometry mapping is sound
- the chapter progression is already present
- the page has the beginnings of a ritual atmosphere

### Current problems
- the sacred geometry viewer dominates the screen
- the principle content is visually subordinate
- the interaction model feels like browsing, not practice
- the page is informative, but not meditative

In short:
the visual hierarchy and contemplative hierarchy are reversed.

## 4. Experience Goal

The page should feel like entering a guided inner chamber.

When someone opens `/journey`, the experience should answer:
- what do I do here?
- how do I begin?
- what should I focus on?
- what should I carry out of the session?

The page should lower cognitive load and increase contemplative depth.

## 5. Recommended Experience Model

The redesigned page should follow this ritual sequence:

1. `Arrival`
- one short orientation passage
- one clear start action
- immediate reduction of clutter

2. `Breath`
- simple breath cue
- restrained motion
- geometry used as a concentration anchor

3. `Principle`
- one principle at a time
- short axiom
- one meditation passage
- one core instruction

4. `Reflection`
- one prompt, not a list
- enough space to pause and remain with the thought

5. `Integration`
- short closing line
- simple next step
- optional deeper study link

## 6. Layout Direction

### Desktop
The page should rebalance the relationship between text and geometry.

Recommended structure:
- primary text and ritual pacing controls in one region
- a contained geometry stage in a second region
- progress or principle navigation in a secondary supporting region

The geometry frame should feel like a ritual object, not a dominant canvas.

### Mobile
The page should be text-first and session-first.

Recommended structure:
- orientation and practice cue first
- smaller contained geometry stage
- simple sticky controls
- no oversized sacred geometry pushing the text below the fold

## 7. Visual Hierarchy

## 8. Content Model Changes

The current principle data is suitable for study, but not enough for guided meditation.

Each principle should gain meditation-specific fields such as:
- `meditationTitle`
- `breathCue`
- `focusLine`
- `meditationBody`
- `reflectionPrompt`
- `integrationLine`
- optional `durationHint`

These fields should complement, not replace, the existing study fields.

## 10. Interaction Model

Primary controls:
- `Begin session`
- `Pause`
- `Next principle`
- `Previous principle`
- `Stillness mode`

Later options:

## 11. Copy Principles

The writing on `/journey` should not read like explanatory site copy.

It should feel:
- slower
- more direct
- more spacious
- more invitational

Copy should instruct attention, not explain theory at length.
